# Situation Room Dashboard — Discovery Interview (Phase 0)

This interview is designed to turn your goals into a concrete v1 build plan for the local `situation_room` site.

## 1) Outcomes & Daily Workflow
1. What does a “great day” with this dashboard look like in 30 seconds, 5 minutes, and 20 minutes?
2. What are the **top 3 decisions** this dashboard should help you make each day?
3. Where will you use it most (desktop, mobile, tablet, wall display)?
4. How often should data refresh by section (real-time, hourly, daily)?

## 2) Priority Modules (Must/Should/Could)
For each area, label **Must / Should / Could** and define your ideal summary card + drill-down detail:
- Sports
- Work (projects, tasks, meetings, deadlines)
- AI news
- Stocks / portfolio watch
- Deloitte posturing (wife’s business intelligence)
- Weather
- Personal development (habits, reading, goals)
- Calendar integration (Apple + Google)

## 3) Sports
1. Which leagues/teams matter most?
2. Do you want only scores + schedules, or also injuries, standings, betting lines, and highlights?
3. What should trigger alerts (e.g., game start in 30 min, close game in 4th quarter)?

## 4) Work / Execution
1. What tools are primary (Notion, Jira, Asana, Linear, Google Docs, Slack, email)?
2. Should we track just deadlines or also progress metrics and blockers?
3. Do you want a daily standup panel autogenerated each morning?

## 5) AI News
1. Preferred sources (OpenAI blog, Anthropic, arXiv categories, X lists, newsletters)?
2. Should content be ranked by impact, novelty, and actionability?
3. Do you want a “brief me in 3 bullets” summary generated each day?

## 6) Stocks & Financial View
1. Which tickers/indices/crypto are essential?
2. Are you tracking watchlist only or brokerage-connected positions?
3. What metrics matter most (price, % change, volume, earnings calendar, analyst revisions)?
4. Risk preference: conservative, balanced, aggressive?

## 7) Deloitte Posturing (Business Intel)
1. What does “posturing” include for your use case (press releases, partner moves, service line launches, hiring trends, M&A)?
2. Which competitors should be compared side-by-side?
3. How should insights appear: timeline, SWOT flags, weekly briefing?

## 8) Weather & Local Context
1. Which locations matter (home, travel cities, client cities)?
2. What weather thresholds should alert (storm, air quality, pollen, heat index)?

## 9) Personal Development
1. What are your active goals (fitness, books, courses, spiritual practices, journaling)?
2. How should progress be scored (streaks, completion %, weekly reflection)?
3. Do you want nudges/reminders, and when?

## 10) Calendar & Identity
1. Which calendars should be connected first (Google, Apple iCloud, both)?
2. Is one-way read enough for v1, or do you need create/update events?
3. Should the dashboard support multiple people/profiles later?

## 11) UX Preferences
1. Preferred layout style: command center grid, left-nav app, or single-scroll briefing?
2. Theme: dark, light, auto?
3. What should the default “first screen” show in the first 10 seconds?

## 12) Privacy, Security, Hosting
1. Is this strictly local-only for now?
2. Any data that must never leave your machine?
3. Acceptable auth flow for OAuth tokens (local encrypted store vs env vars)?

## 13) Technical Constraints
1. Preferred stack for local site (Next.js, Vite React, Svelte, etc.)?
2. Should we design for eventual cloud deploy, or keep fully local architecture?
3. Do you want plugin-style modules so you can add/remove domains quickly?

## 14) v1 Success Criteria
Define v1 as done when:
- [ ] Dashboard opens in <2 seconds locally.
- [ ] Top priority modules load reliably.
- [ ] Daily summary appears in one glance.
- [ ] Calendar sync works for at least one provider.
- [ ] At least 3 useful alert rules are live.

---

## Proposed Build Sequence (after interview)
1. **Finalize requirements + rank priorities**.
2. **Design data contracts per module** (inputs, transforms, outputs).
3. **Set up local web shell** (auth, layout, module framework).
4. **Integrate first 3 Must-have modules**.
5. **Add calendar OAuth connector(s)**.
6. **Add alerts + daily briefing orchestration**.
7. **Polish UX + harden privacy/security defaults**.

## MCP Status (current environment)
No MCP resources are currently available in this workspace session. If MCP servers are added (calendar, portfolio, project tools, news APIs), we can wire those as first-class connectors in the module layer.
